Welcome to Nightshade

This page is to test use cases and specifically how Chrome is handling aria-hidden for focusable elements. Here are a few examples of common development patterns I see when testing sites for accessibility. This is not an all inclusive list and is likely to evolve over time. We need to also test if screen readers are able to encounter the elements with aria-hidden on first TAB press, or after the element has been added to the accessibility tree.

Test Cases

  • Decorative icons hidden from screen-readers inside link elements (icon fonts)
  • Decorative icons hidden from screen-readers inside link elements (svg icons)
  • Redundant links (graphic links immediately followed by link text that point to same destination)
  • Button elements
  • Carousel slides (dynamic aria-hidden for slides that are no longer in visible)
  • Visually Hidden Nested/Collapsed Content
